# Phase 1.5 — MongoDB Sync, Labeled Fallback, STT Upgrades

**Date:** 2026-07-06 · **Parent:** [Phase 1 spec](2026-07-06-phase1-oracle-design.md) · **Status:** approved in conversation

## Goal

Make the Oracle actually useful with the user's real data (18 demo records in
MongoDB Atlas `aixlab.demos`), stop dead-ending on off-library questions, and
make speech recognition faster and domain-aware.

## Decisions (locked)

1. **MongoDB = source of truth; LanceDB = local index.** New `reachy-vec
   sync-mongo` reads `aixlab.demos` via `MONGODB_URI` (from `.env`), formats
   each record (title, project, authors, tags, URL, note) into text, chunks +
   embeds with the existing BGE pipeline, and **replaces** all mongo-sourced
   rows (idempotent). The pre-existing 1536-dim `embedding` field in Mongo is
   ignored — one local embedding space. `aixlab.users` is never read.
   Spoken-friendly source string: `demo: <title>`.
2. **Score-gated labeled fallback.** Doc search gains cosine scores. If the
   best hit is below `rag_min_score` (default 0.5, `REACHY_VEC_RAG_MIN_SCORE`),
   the LLM answers from general knowledge with the spoken prefix
   "Not from our team docs, but..." and no sources. Above threshold: grounded
   answer, unchanged.
3. **STT:** default model `base.en` (benchmarked 0.43s vs 0.67s `base`,
   correct transcript; `large-v3-turbo` measured 5.04s on CPU — rejected).
   Whisper gets an `initial_prompt` built from KB demo titles at startup
   (domain-vocabulary priming). Optional `stt_backend="openai"`
   (`gpt-4o-transcribe`) for maximum accuracy at ~1s network cost; default
   stays `"local"`.

## Error handling

- `sync-mongo` with no/unreachable `MONGODB_URI`: clear message, exit 1;
  never partial-deletes (delete old rows only after new embeddings computed).
- OpenAI STT failure: log + return None (heard nothing) rather than crash.

## Out of scope

Live Mongo querying, Atlas Vector Search, fish-speech TTS, Phase 2 features.
