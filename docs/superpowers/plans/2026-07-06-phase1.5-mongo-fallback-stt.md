# Phase 1.5: Mongo Sync + Labeled Fallback + STT Upgrades — Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** `reachy-vec sync-mongo` indexes the aixlab demo library; RAG answers off-library questions with a labeled general-knowledge fallback; STT is faster (base.en), domain-primed, with an optional OpenAI backend.

**Tech:** pymongo, existing BGE/LanceDB pipeline, faster-whisper, openai.

## Global Constraints

- Tests: no network, no Mongo, no models, no devices — fakes only.
- New settings: `rag_min_score: float = 0.5`, `stt_backend: str = "local"`; `stt_model` default changes to `"base.en"`.
- Mongo credentials only via `MONGODB_URI` env/.env; never in code or logs.
- Commit per green cycle.

### Task 1: Scored doc search + source-prefix delete (store)

- `Store.search_docs_scored(vector, k=5) -> list[tuple[DocChunk, float]]` (cosine similarity, high = similar; use `.metric("cosine")`, score = 1 − distance)
- `Store.delete_docs_by_source_prefix(prefix: str) -> None`
- Tests: scored search returns ~1.0 for exact vector; delete removes only matching-prefix rows.

### Task 2: Mongo sync (store/mongo_sync.py + cli)

- `format_demo(doc: dict) -> str` — title/project/authors/tags/URL/note lines; tolerate missing fields.
- `sync_demos(demos: list[dict], store, embedder) -> int` — format → chunk_text → embed → build rows (`source=f"demo: {title}"`, `chunk_id=f"mongo:demos/{_id}:{i}"`) → `delete_docs_by_source_prefix("demo: ")` → add. Returns chunk count.
- `cli/sync.py`: `sync-mongo` command — load_dotenv, read `MONGODB_URI` (exit 1 with message if unset), fetch `aixlab.demos` via pymongo, call `sync_demos`, echo count. Register in `cli/__init__.py`.
- Tests: format_demo content + missing fields; sync idempotency (run twice → same count, no duplicates) with FakeEmbedder; keeps non-demo rows.

### Task 3: Labeled fallback (brain/rag.py)

- `answer()` uses `search_docs_scored`; if no hits OR best score < `settings.rag_min_score`: fallback prompt (general knowledge, must start answer with "Not from our team docs, but"), `sources=[]`. Empty-KB message only when doc_count == 0.
- Tests: strong match → grounded (unchanged); weak score → fallback prefix instruction in prompt, empty sources; monkeypatched threshold.

### Task 4: STT upgrades (audio/listen.py + config + run wiring)

- `stt_model` default → `"base.en"`.
- `MicTranscriber(initial_prompt: str | None = None)`; passed to `whisper.transcribe(..., initial_prompt=...)`.
- `OpenAITranscriber(client, initial_prompt=None)` — same VAD capture path (share a `_capture_utterance` helper), encodes WAV in-memory, `client.audio.transcriptions.create(model="gpt-4o-transcribe", ...)`; failure → log + None.
- `make_transcriber(client=None) -> Transcriber` selecting by `settings.stt_backend`.
- `Store.demo_titles(limit=30) -> list[str]` (distinct `source` values stripped of `demo: ` prefix) → `run.py` builds `initial_prompt = "Vocabulary: " + ", ".join(titles)` and uses `make_transcriber`.
- Tests: prompt plumbing (fake whisper via injected transcribe fn is overkill — test `make_transcriber` selection + `OpenAITranscriber` transcription-call shape with a fake client + demo_titles).

### Task 5: Manual smoke

`uv run reachy-vec sync-mongo` → "Synced N chunks from aixlab.demos." →
`uv run reachy-vec chat` → ask "what demos do we have about food insecurity?" (grounded, cites `demo: Liverpool City Food Mapping`) and "what is the capital of France?" (spoken fallback prefix). Then `reachy-vec run` end-to-end.
